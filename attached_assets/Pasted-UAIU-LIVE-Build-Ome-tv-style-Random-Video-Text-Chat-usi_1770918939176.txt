WHAT TO BUILD (MVP that works):
1) Matchmaking like Ome.tv (no time limit)
- When user clicks Start, they enter a waiting queue.
- When a second user clicks Start, pair them immediately.
- Create a private Daily room server-side and issue meeting tokens server-side.
- Send BOTH users the same {matchId, roomUrl, token} in real time.
- Session continues indefinitely until user clicks Next/End (NO 1-minute timer or auto end).

2) Text chat always works (even if camera/mic denied)
- On match, enable a live chat panel with message list + input + send.
- Messages send over Socket.IO and are delivered to the matched partner instantly.
- If video fails or permissions are denied, keep the match alive and keep text chat working.

3) Video+audio via Daily Prebuilt embed
- Embed Daily Prebuilt in the match view using daily-js createFrame().
- Join room using roomUrl + token (token should be passed to prebuilt join).
- Provide UI buttons: Next, End, mute, camera toggle (can be minimal; Daily prebuilt controls OK, but Next/End must be ours).

4) Ome.tv-like UI flow
- Start screen: Start button.
- Waiting: “Looking for someone…” + Cancel.
- Matched screen: Video area + Chat panel + Next button prominent + status text.
- Show partner disconnected message if the other user leaves.

BACKEND REQUIREMENTS:
- Use Node/Express (or whatever server exists) + Socket.IO.
- Implement an in-memory queue + match map:
  - waitingQueue: array of socketIds
  - matches: map matchId -> {aSocketId, bSocketId, roomUrl}
- Events:
  - match:start (client -> server) enter queue or pair
  - match:cancel (client -> server) remove from queue
  - match:next (client -> server) leave match and requeue
  - match:leave (client -> server) end match and notify partner
  - chat:send (client -> server) broadcast to partner
  - match:found (server -> client) contains {matchId, roomUrl, token, partner}
  - chat:message (server -> client) contains {from, text, ts}
  - match:partner_left (server -> client)

DAILY API DETAILS (server-side only):
- Create room:
  POST https://api.daily.co/v1/rooms
  body: { name: "<matchId>", privacy: "private" }
  header: Authorization: Bearer ${DAILY_API_KEY}
- Create meeting token (one per user):
  POST https://api.daily.co/v1/meeting-tokens
  body: { room_name: "<matchId>", user_name: "<username>", is_owner: false }
- Room URL must be:
  https://uaiulive.daily.co/<matchId>
